Overview

This SOP covers the complete process of integrating a new tenant into the property management system — from collecting their information to verifying all communication channels are working. Follow each phase in order.

Estimated time per tenant: 5–10 minutes (setup) + verification once tenant responds

Tools needed: Dashboard access, tenant's contact information

Phase 1 — Gather Tenant Information

BEFORE YOU START

Collect the following details for each tenant. All fields are important for the system to identify tenants across channels.

Phone NumberWith country code (e.g. 15145551234)YesUsed for SMS, WhatsApp, and phone call identification
Emailtenant@email.comYesUsed for email channel identification + vendor dispatch notifications

Telegram IDNumeric (e.g. 6274604148)No*Auto-captured when tenant messages the bot

* Telegram ID: You do NOT need to collect this manually. The system captures it automatically when the tenant sends their first message to the Telegram bot. Leave this field blank during setup.

Phone number format: Always include the country code WITHOUT the + sign. For Canadian numbers: 1 + area code + number. Example: 15145551234 (not +1-514-555-1234 or 514-555-1234).

This exact format is critical — the system uses it to match incoming SMS, WhatsApp messages, and phone calls to the correct tenant.

Phase 2 — Add Tenant to the Dashboard

DASHBOARD
  1. Log in to the dashboard Open julia.srv1285597.hstgr.cloud in your browser.
  2. Make sure the property exists first Go to Properties in the sidebar. If the tenant's building is not listed, click Add Property and enter the property name and full address. Save it before proceeding.
  3. Go to the Tenants page Click Tenants in the sidebar.
  4. Click "Add Tenant" The button is at the top right of the page.
  5. Fill in the tenant details • Name: Full name (First Last)
    • Phone: With country code, no spaces or dashes (e.g. 15145551234)
    • Email: Tenant's email address
    • Unit Number: Their unit (e.g. 203)
    • Property: Select from the dropdown
    • Telegram ID: Leave blank — it will be filled automatically
  6. Click Save The tenant now appears in the Tenants list, grouped under their property.

Common mistakes to avoid:

• Do NOT add the + sign to phone numbers — just digits (15145551234)

• Do NOT leave the phone blank — it's needed for SMS, WhatsApp, and phone identification

• Do NOT add the same phone number for two different tenants — the system uses it as a unique identifier

• Do NOT manually enter a Telegram ID — let the system capture it automatically

Phase 3 — Send the Welcome Email

COMMUNICATION
  1. Open the welcome email template Use the template provided (English and French versions available). Choose the appropriate language for each tenant.
  2. Personalize the email • Replace [Owner Name] with your name
    • Replace [Company Name] with your company name
    • Optionally add the tenant's name in the greeting ("Dear [Tenant Name]" instead of "Dear Tenant")
  3. Send from your own email The email should come from you (the landlord), not from a third party. Tenants trust communication from their landlord.
  4. Confirm the Telegram link works Before sending, click the Telegram bot link in the email yourself to verify it opens correctly: t.me/julia_pm_bot

Tip — send in batches: If you have many tenants, consider sending to one building at a time. This lets you handle any questions before moving to the next group.

Phase 4 — Telegram Activation

TENANT ACTION REQUIRED

This is the only step that requires action from the tenant. When a tenant clicks the Telegram link and sends their first message, the system automatically:

  1. Tenant opens the link They click t.me/julia_pm_bot from the welcome email (or search for @julia_pm_bot in Telegram).
  2. Tenant sends any message Even a simple "Hello" works. The system detects them by their Telegram account.
  3. System captures their Telegram ID The tenant's Telegram ID is automatically saved to their record in the database. You do not need to do anything.
  4. Bot responds The AI assistant greets them and asks how it can help. The tenant is now fully set up on Telegram.

What if a tenant doesn't have Telegram?

That's fine — Telegram is optional. They can still use WhatsApp, SMS, Email, or Phone. All channels work independently. However, you should encourage Telegram or WhatsApp as they provide the best experience (instant replies, photo support, interactive buttons).

What if a tenant messages but isn't recognized?

If a tenant's phone number or email doesn't match any record in the system, the bot will not respond. Double-check that:

• Their phone number in the dashboard matches exactly (same digits, with country code)

• Their email is spelled correctly

• They are messaging from the correct phone/account

Phase 5 — Verify All Channels

VERIFICATION

After the tenant has been added and received the welcome email, verify that the system recognizes them on each channel they'll use.

Quick Verification Checklist

  • Telegram: Tenant sent first message → bot responded → Telegram ID saved in dashboard
  • WhatsApp: Tenant sent a WhatsApp message to +1 438-900-9998 → bot responded
  • SMS: Tenant texted +1 438-900-9998 → bot responded via SMS
  • Email: Tenant emailed from their registered address → bot responded via email
  • Phone: Tenant called +1 438-900-9998 → voice agent greeted them by name
  • Dashboard: Tenant appears under correct property with phone, email, and Telegram ID filled in
  • Notification: You received a Telegram notification when the test ticket was created

You don't need to test all 5 channels for every tenant. If Telegram and one other channel work, the tenant is properly integrated. The important thing is that their phone and email are correct in the system.

How to Confirm Telegram ID Was Captured

  1. Go to Tenants on the dashboard Find the tenant in the list.
  2. Click Edit on their record The Telegram ID field should now show a number (e.g. 6274604148). If it's blank, the tenant hasn't messaged the bot yet.

Bulk Onboarding — Multiple Tenants

When onboarding an entire building or portfolio, follow this workflow:

  1. Prepare a list Gather name, phone (with country code), email, unit number, and property for every tenant. A simple spreadsheet works.
  2. Add the property first Make sure the property exists on the dashboard before adding any tenants.
  3. Add tenants one by one Go to Tenants → Add Tenant for each person. Double-check phone number format each time.
  4. Send the welcome email to all tenants Use BCC if sending to multiple tenants at once so they don't see each other's email addresses. Or send individually for a more personal touch.
  5. Track who has activated Telegram Over the next few days, check the Tenants page. Tenants who have messaged the bot will have their Telegram ID filled in. Follow up with those who haven't.
  6. Send a reminder after 3–5 days For tenants who haven't activated Telegram, send a short follow-up: "Just a reminder — you can reach us anytime on Telegram for fast maintenance support. Click here to get started: t.me/julia_pm_bot"

Troubleshooting

ProblemLikely CauseSolution
Tenant messages bot but gets no reply Phone/email doesn't match any record Check the tenant's phone number format on the dashboard. Must include country code, digits only.
Telegram ID is blank after tenant messaged Tenant messaged from a different Telegram account, or system didn't match them Ask tenant to try again. If still blank, have them send /start to @julia_pm_bot.
Tenant gets reply on Telegram but not SMS SMS uses phone number for lookup — might be formatted differently Verify the phone number in the dashboard matches exactly what Twilio receives (digits with country code).
WhatsApp messages not going through Tenant may be messaging the wrong number, or not using WhatsApp Confirm they're messaging +1 438-900-9998 via WhatsApp (not regular SMS).
Voice agent doesn't greet tenant by name Phone number mismatch (caller ID vs. stored number) Check if the caller ID includes country code. Some phones show 5145551234 while DB has 15145551234.
Duplicate tenant records Same person added twice with slightly different info Delete the duplicate on the Tenants page. Keep the one with the Telegram ID filled in.
Tenant's email replies aren't picked up Sending from a different email than the one in the system Update the tenant's email on the dashboard to match what they're sending from.

Removing a Tenant

When a tenant moves out:

  1. Resolve or close any open tickets Go to Tickets → find their open tickets → Mark Resolved on each one.
  2. Delete the tenant record Go to Tenants → find the tenant → click Delete → confirm. The bot will stop responding to their messages.

Note: Deleting a tenant does NOT delete their past tickets or message history. Those remain in the system for your records. Only the tenant's profile is removed, which stops the bot from recognizing future messages from that person.
